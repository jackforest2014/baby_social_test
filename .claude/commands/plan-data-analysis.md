# 数据分析规划

启动数据分析规划流程，根据 PRD 设计埋点方案、指标体系和数据看板。

## 执行步骤

### 1. 确认 PRD

- 读取 `docs/prd/` 目录下的 PRD 文档
- 如果用户指定了具体文档，使用指定的文档
- 如果未指定，列出可用的 PRD 文档供用户选择

### 2. 分析已有数据基础

- 扫描项目中已有的埋点实现
- 检查是否已接入分析平台
- 了解已有的日志和监控体系

### 3. 设计数据方案

使用 `data-analyst` Agent 进行完整的数据分析规划：

- 业务目标拆解为可量化指标
- 埋点事件体系设计（事件名、属性、触发时机、植入位置）
- 指标体系设计（L1 业务指标 / L2 功能指标 / L3 技术指标）
- 数据看板设计（产品看板 / 运营看板 / 技术看板）
- 前后端埋点实现建议
- 数据隐私合规检查
- A/B 测试支持方案

### 4. 生成报告

- 使用模板 `.claude/templates/data-analysis-plan.md.tpl` 生成报告
- 保存到 `docs/analytics/{subject}.md`

## 输入参数

- `$ARGUMENTS`：可选，指定 PRD 文档路径或主题名。如未提供则交互式选择。

## 输出

- 数据分析规划报告：`docs/analytics/{subject}.md`
- 包含埋点清单、指标体系、看板设计和实现建议

## 后续操作

报告生成后，建议：
1. 用户审阅埋点方案和指标定义是否合理
2. 将报告作为输入，在 `/design-backend` 和 `/design-frontend` 中纳入数据采集相关的技术设计
