# 代码现状侦察

启动代码侦察流程，分析当前项目中与指定 PRD 相关的已有实现。

## 执行步骤

### 1. 确认 PRD

- 读取 `docs/prd/` 目录下的 PRD 文档
- 如果用户指定了具体文档，使用指定的文档
- 如果未指定，列出可用的 PRD 文档供用户选择

### 2. 启动侦察

使用 `code-scout` Agent 进行全面扫描和分析：

- 需求关联分析：将 PRD 功能点映射到代码中的已有实现
- 架构分析：已有的模块划分、分层结构、可复用基础设施
- 接口分析：已有的 API 接口及其与新需求的关系
- 代码分析：后端和前端的已有实现
- 中间件分析：PostgreSQL、Redis 等的使用现状
- 第三方服务分析：已集成的外部服务
- 风险识别：兼容性、数据迁移、接口变更、性能、安全等

### 3. 生成报告

- 使用模板 `.claude/templates/situation-report.md.tpl` 生成现状分析报告
- 保存到 `docs/situation/{subject}.md`

## 输入参数

- `$ARGUMENTS`：可选，指定 PRD 文档路径或主题名。如未提供则交互式选择。

## 输出

- 现状分析报告：`docs/situation/{subject}.md`
- 包含需求覆盖矩阵、架构现状图、风险清单和改造建议

## 后续操作

报告生成后，建议：
1. 用户审阅报告中标注的风险和注意事项
2. 将报告作为输入，执行 `/design-backend` 或 `/design-frontend` 进行技术方案设计
