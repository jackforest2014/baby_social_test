# 用户故事扩展与场景覆盖

基于已有的 PRD 文档和初始用户故事，系统性地扩展场景覆盖，确保测试充分性。

## 触发时机

PRD 文档完成且经用户确认后，建议执行此技能。

## 输入

- PRD 文档（`docs/prd/` 目录下的相关文件）
- 初始用户故事（如已有）
- 业务规则文档（如 skill_base.md 等）

## 执行步骤

### 第 1 步：分析现有覆盖

1. 读取相关 PRD 文档，提取核心功能点和业务规则
2. 读取已有的用户故事（如有），分析当前覆盖的场景类型
3. 识别覆盖空白：哪些角色、意图、边界条件、异常场景尚未覆盖

### 第 2 步：设计场景分类体系

1. 根据业务特征，将用户故事归入若干意图/场景分类
2. 确保分类体系覆盖以下维度：
   - **正常业务流程**：各类用户的典型使用场景
   - **边界与异常**：极端输入、异常行为、错误操作
   - **安全与攻击**：注入攻击、身份伪装、社会工程学
   - **多语言/多文化**：不同语种、方言、文化背景
   - **特殊人群**：老年人、未成年人、残障用户等
   - **对话行为异常**：超长消息、沉默、连续发送、话题跑偏等
3. 输出分类索引文档

### 第 3 步：扩展用户故事

1. 基于分类体系，为每个空白分类生成用户故事
2. 每个故事包含：场景描述、角色画像、模拟交互（至少 6 轮）、测试维度、最大风险点
3. 确保故事之间不重复不相似，每个故事有独特的测试挑战
4. 批量生成时使用并行 agent 提高效率（建议每批 15-25 个）

### 第 4 步：生成验收标准

1. 为每个用户故事生成可测试的验收标准
2. 验收标准需包含：意图识别准确性、边界守护要求、信息收集完整性
3. 按分类组织验收标准文档

### 第 5 步：汇总与交付

1. 更新分类索引文档（含场景数量统计）
2. 在相关 PRD 文档中链接验收标准文档
3. 更新进度记录

## 输出

- 扩展后的用户故事文件（每个故事一个独立 .md 文件）
- 场景分类索引文档（按分类汇总所有故事）
- 验收标准文档（按分类组织的验收条目）

## 文件规范

- 用户故事保存路径：由用户指定或沿用已有目录
- 文件命名格式：`{编号}. {英文简称}.md`
- 每个文件必须包含目录
- 分类索引和验收标准文档保存在用户故事同级目录或 PRD 目录下
